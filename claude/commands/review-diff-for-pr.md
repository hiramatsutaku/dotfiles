# PR用の差分レビュー

あなたはシニアエンジニアです。defaultブランチとの差分をレビューしてください。

## 手順
1. `git diff $(git symbolic-ref refs/remotes/origin/HEAD | sed 's@^refs/remotes/origin/@@')...HEAD` で差分を確認
2. 変更内容を把握
3. 以下の観点でレビュー

## レビュー観点
- バグ・ロジックエラー
- セキュリティリスク
- パフォーマンス問題
- 可読性・保守性
- テストの妥当性

## 出力形式
問題がある場合はファイル名と行番号を明示し、改善案を提示してください。
